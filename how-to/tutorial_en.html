How Can I Add My Favorite Location?

Preface

We use data from OpenStreetMap for Veggiekarte. This has many advantages, for example, anyone can participate and add or correct information. And when you add new entries for Veggiekarte, not only do users of our site benefit, but also those of many other OpenStreetMap applications.

1. Prerequisite

If you don't have an OpenStreetMap account yet, you can easily create one on this page.

2. Editing

After you've signed up for OpenStreetMap, go to the location of interest on the OpenStreetMap website and click on the green "Edit" button at the top left. This will take you to the editor, which runs directly in the browser. If you've never edited before, you might want to take a look at the introduction offered there, otherwise click on "Edit now."

Now there are two cases:

2.1. The location is already in the database

You can tell because it's already displayed on the map (as a point or area).

Click on the corresponding object on the map that represents the location. In the left part, you should then see a blue menu item titled "Properties (x)," click on it. With the large plus sign, you can now add new properties to the object. In the left area of the new row, you enter the so-called key, and in the right area, the value.

The most important keys for us are diet:vegan and diet:vegetarian. These keys determine whether a location is displayed on Veggiekarte or not. Here's an overview of the values that are relevant to us:

Key Value Description
diet:vegan only for locations whose offerings are completely vegan
yes for locations that also have vegan options
limited for locations that have vegan options but do not clearly mark them
diet:vegetarian only for completely vegetarian locations
yes for locations that also have vegetarian options

Once you've made your entries, just click on "save" at the top and everything is done. That wasn't so hard, was it?

Your changes should be visible on Veggiekarte within an hour. Thank you very much!

2.2. The location is not yet in the database

So it's not yet displayed on the map.

Compared to point 2.1., a few small preliminary steps are necessary.

Select "Point" in the editor menu at the top and then click on the location on the map where the location is. Then select what it is about on the left. 'Caf√©', 'Restaurant' or similar already exist as options. Definitely fill in the name and preferably also the address. Then you can proceed as in point 2.1.

3. In addition

3.1. Maintain Additional Information

You are welcome to add further keys as additional information to be displayed on Veggiekarte:

  • cuisine Type of offered dishes
  • addr:* Address
  • opening_hours Opening hours
  • contact:phone Phone number (in international format)
  • contact:email Email address
  • contact:website Website
  • contact:facebook Facebook page
  • contact:instagram Instagram page

3.2. Increase Data Quality

Since OpenStreetMap is a huge community project, it happens that data is entered incorrectly or becomes outdated. To improve data quality, we have created another map and a table where you can find places where problems have been identified.

Not all "problems" listed there need to be true. So please always carefully check the facts before making a change.

If you have ideas for further checks, feel free to let us know (e.g. by opening an issue on Codeberg).

3.3. Improve OpenStreetMap

And perhaps you'd like to improve even more on the map, the possibilities are very diverse. An overview of common tags is also provided by the OpenStreetMap Wiki. There is also a German forum available.
